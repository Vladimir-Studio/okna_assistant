# ИИ-ассистент для поддержки и FAQ компании «Континент Окна Крым»

RAG-ассистент для сайта: база знаний → FAISS → FastAPI → виджет чата.

За счёт автоматизации ответов через ИИ-ассистента среднее время ответа отдела поддержки уменьшилось на 25%.

---

## Какие бизнес-задачи закрывает

### Снижение нагрузки на первую линию поддержки
Большинство обращений клиентов — повторяющиеся вопросы: стоимость, сроки, замер, гарантия. Ассистент обрабатывает их автоматически, не привлекая сотрудников. Операторы получают только нестандартные запросы, требующие живого участия.

### Автоматизация обработки типовых обращений
Ассистент мгновенно отвечает на вопросы по базе знаний компании — без скриптов, без ожидания, без человеческого фактора. Новые темы подключаются за минуты: достаточно добавить `.txt`-файл или запись в `faqs.json` и пересобрать индекс.

### Повышение скорости и качества взаимодействия с клиентами
Ответ формируется за 1–3 секунды на основе актуальной базы знаний. Клиент получает точный, структурированный ответ на русском языке — без лишних слов и без риска получить устаревшую информацию. Встроенная система оценок (👍/👎) и метрики позволяют постоянно улучшать качество ответов на основе реальных данных.

### Доступность поддержки 24/7 без привлечения людей
Ассистент работает круглосуточно без выходных и праздников. Клиент получает ответ в любое время — ночью, в выходной, в нерабочие часы — без задержек и без необходимости ждать открытия офиса.

## Системный промпт

Поведение ассистента задаётся системным промптом в `backend/app.py`. Он состоит из четырёх частей:

**Цель** — ассистент представляется как сотрудник конкретной компании («Континент Окна Крым»), что сразу задаёт контекст и исключает ответы «вообще».

**Стиль** — дружелюбно-профессиональный, кратко и по делу, только русский язык. Клиент получает ощущение живого общения без канцелярита и лишних слов.

**Сценарии** — модель явно инструктирована использовать только переданный FAQ-контекст. Если ответа в контексте нет — предложить связаться с поддержкой, а не придумывать.

**Ограничения** — одной строкой: не выдумывай факты · не выходи за контекст · не давай гарантий · не уходи от темы компании.

Чтобы скорректировать поведение ассистента — отредактируйте переменную `system_prompt` в функции `chat()` в файле `backend/app.py`.

## Логирование диалогов и метрики качества

Каждый диалог автоматически записывается в `data/dialogs.jsonl` — один JSON-объект на строку.

Пример записи:
```json
{
  "dialog_id": "uuid",
  "ts": "2026-03-31T10:00:00+00:00",
  "message": "Сколько стоит замер?",
  "answer": "Замер бесплатный...",
  "context_questions": ["Бесплатный ли замер?"],
  "retrieval_scores": [0.21, 0.45, 0.67],
  "avg_retrieval_score": 0.443,
  "feedback": 1
}
```

`retrieval_scores` — FAISS L2-дистанции для каждого найденного документа. Чем меньше — тем точнее совпадение с вопросом пользователя. Используйте `avg_retrieval_distance` из `/metrics` чтобы подобрать порог `RAG_DISTANCE_THRESHOLD` в `app.py`.

`feedback` заполняется через `POST /feedback` или кнопки 👍/👎 в виджете чата.

Просмотр логов вручную:
```bash
type data\dialogs.jsonl
```

Экспорт в CSV для анализа:
```bash
python -c "import pandas as pd; pd.read_json('data/dialogs.jsonl', lines=True).to_csv('dialogs.csv', index=False)"
```

Агрегированная статистика доступна в браузере: `http://localhost:8000/metrics`

## Быстрый старт

### 1. Окружение

```bash
pip install -r requirements.txt
```

Создайте `.env` в корне проекта:

```env
OPENAI_API_KEY=ваш_ключ_openai
OPENAI_MODEL=gpt-4.1-mini
TEMPERATURE=0.2
MAX_TOKENS=300
```

### 2. База знаний

Поддерживаемые форматы:
- `data/faqs.json` — массив объектов `{ "question": "...", "answer": "..." }`
- `data/*.txt` — текстовые документы (первая строка становится заголовком/вопросом, остальное — ответом)

Чтобы добавить или обновить базу знаний:
1. Добавьте/отредактируйте файлы в папке `data/`
2. Пересоберите индекс:

```bash
python -m backend.build_index
```

3. Перезапустите сервер.

### 3. Запуск сервера

```bash
uvicorn backend.app:app --reload --host 0.0.0.0 --port 8000
```

Откройте `http://localhost:8000` — виджет чата доступен сразу.

## Эндпоинты

| Метод | Путь | Описание |
|-------|------|----------|
| `GET` | `/` | Фронтенд (index.html) |
| `POST` | `/chat` | Отправить вопрос, получить ответ |
| `POST` | `/feedback` | Оценить ответ (👍/👎) |
| `GET` | `/metrics` | Статистика диалогов |
| `GET` | `/health` | Проверка статуса |
| `GET` | `/docs` | Swagger UI |

### POST /chat

Запрос:
```json
{ "message": "Сколько стоит замер?", "top_k": 3 }
```

Ответ:
```json
{
  "answer": "Замер бесплатный...",
  "context": [{ "question": "...", "answer": "...", "source": "doc1.txt" }],
  "dialog_id": "uuid"
}
```

### POST /feedback

```json
{ "dialog_id": "uuid", "feedback": 1 }
```

`feedback`: `1` = хорошо, `-1` = плохо, `0` = нейтрально.

### GET /metrics

```json
{
  "total": 42,
  "with_feedback": 10,
  "feedback_positive": 8,
  "feedback_negative": 2,
  "satisfaction_rate": 0.8,
  "avg_retrieval_distance": 0.312,
  "recent": [...]
}
```

## Безопасность и качество

- Входящие сообщения обрезаются до 500 символов, очищаются от управляющих символов
- RAG validation: если минимальная FAISS-дистанция превышает порог `RAG_DISTANCE_THRESHOLD` (задаётся в `app.py`), GPT не вызывается — пользователь получает fallback-ответ. Порог подбирается по значению `avg_retrieval_distance` в `/metrics`
- `max_tokens` ограничивает длину ответа
- Пользовательский ввод изолирован от системного промпта

## Как это работает

1. `build_index.py` читает `faqs.json` и все `.txt` из `data/`, считает эмбеддинги через `text-embedding-3-small`, сохраняет FAISS-индекс
2. На запрос `/chat` backend считает эмбеддинг вопроса, ищет ближайшие записи в FAISS
3. Найденный контекст передаётся в `gpt-4.1-mini`, модель генерирует ответ на русском
4. Диалог логируется в `data/dialogs.jsonl` с FAISS-дистанциями
5. Пользователь может оценить ответ через кнопки 👍/👎 в виджете
